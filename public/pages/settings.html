🔐 Role-Based Permissions

Understanding access levels and permissions for each user role

Administrator Role

ADMIN

Full System Access

Administrators have complete control over the system with no restrictions.

✅ All Pages Accessible:
  • Analytics Dashboard
  • All Commits (system-wide)
  • All Users Management
  • All Projects
  • Alerts & Monitoring
  • Daily Commits Reports
  • AI Models Configuration
  • Tools Management
  • System Settings
🛠️ Administrative Actions:
  • Create, edit, and delete users
  • Modify system configurations
  • Analyze any commit
  • Configure AI models
  • Generate reports
  • Access all project data

Standard User Role

USER

Limited Read-Only Access

Standard users have restricted access focused on their own contributions.

✅ Visible Pages (Read-Only):
  • Commits - Only see their own commits
  • Profile - Only access their own user details
  • Projects - Only see projects where they're contributors
  • Daily Commits - Can view daily commit reports
❌ Hidden/Blocked Pages:
  • Analytics Dashboard
  • Alerts & Monitoring
  • AI Models Configuration
  • Tools Management
  • System Settings
🚫 Restrictions:
  • Cannot create, edit, or delete any data
  • Cannot view other users' commits
  • Cannot access system configurations
  • Cannot modify user profiles

Note: Role permissions are enforced at both the frontend interface and backend API levels for maximum security. Users cannot bypass restrictions by directly accessing URLs or API endpoints.
